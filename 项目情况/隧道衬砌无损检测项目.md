# 项目背景

隧道衬砌无损检测系统研发从 2023 年 6 月一直到 2025 年 9 月，主要是铁路隧道衬砌经过长时间的使用之后会出现各种病害，典型的如裂缝、渗水、掉块，传统的隧道检测方法是通过人工搭建高架的方式进行逐一排查，这非常的耗费人力物力。同时，由于高铁的天窗期通常只有 3h 甚至更少，这会导致一个隧道会需要多个天窗期进行检查。

# 项目难点

1. **产品画像确定**：从 0 开始，需要构建设备画像、初步选项、技术路线敲定。
2. **时间要求高**：数据量巨大，一条 4km 的隧道就会产生 12 × 40000 的数据，如何将输出结果的时间压缩到相应的范围呢。
3. **确认检测标准**：无参照先例，业主需求模糊。

# 工作内容

## 硬件选项

项目基础构思结合市面上优秀的其他厂家，产品画像是一款轨道小车，然后在其上安装高精度摄像头，通过拍照获取衬砌表观的高精度影像然后通过深度学习算法进行检测。铁路局要求隧道衬砌的裂缝精度要检测到 0.2mm，因此就市面上一些设备进行相应的筛选。先后使用了线阵和面阵相机，最后敲定了 8k 的面阵相机进行数据采集，精度达到 0.15mm/pix。

## 工作流与算法选择

### 工作流

结合各个模块的耗时工作，最后敲定整体工作量如下：

```text
                                               |--------------------------->(pdx,pdy) -------------↓
数据采集 → 数据备份 → 数据拷贝机房 → 图像拼接 → 640图像 → 检测模型(m) → 每图像的json ---------------→ 病害拼接 → 指标计算（掉块渗水面积，裂缝的宽度长度）→ 绘制病害到图像上并输出报告。
                                   |
                                   |--------------------> 分割数据----->每个图像的json  ------------↑
```

### 我的工作

除了参与工作流的指定，在工作流中的检测 + 后处理（病害拼接 + 指标计算 + 报告输出）由我处理。使用 YOLO V5 进行的模型检测，主要包含：数据收集与标记、模型训练与优化。

#### 数据收集与标定

1. 每次采集都会有大量的图片数据产生，利用 YOLO 模型进行数据检测，初筛一次数据。
2. 对初筛的数据标签进行标签的修订，然后做一个数据集，微调一个本次数据的模型来。
3. 用 2 中的模型对数据进行一次细致筛查，两次的交集，然后进行标签修订。
4. 重复步骤，到几乎无新增。

#### 模型训练与优化

1. 最开始并没有用 640 进行验证，而是测试了多种分辨率，因为 640 会导致小的病害消失掉。后续在多方沟通协商后，确定过小的病害不用考虑（工程方面的考虑，如果很小的都检测，会导致病害数量骤然增加，诸多不便）。
2. 测试不同超参数对于模型结果的影响（主要修改了 LR 以及不同的 cls、box、obj、iou 权重）。
3. 数据增强，使用了 YOLO V5 自带的增强外还使用 Copy-Paste 进行贴图增强（在更多的正常墙面进行采样进行离线增强，提升数据量）。
4. 增加干扰类，由于隧道中环境复杂，会有很多干扰，只单纯的让模型学习将至成为背景，成本过高，因此设定了一些常见的干扰类，给模型一个强监督信号（如灯牌、线缆、人员、反光条等），可以有效的降低误分。

#### 后处理

1. 使用图像拼接中输出的每一个图像的 pdx、pdy 从 640 尺度还原到 8000×9000 尺度并且进行病害融合（跨图像病害）。
2. 最开始想的是做一个极大的 mask，然后将病害绘制然后逐项求索，但是内存不允许。
3. 后续直接使用 poly 进行判断，主要数据流如下：

   ```text
   DK+SS(640) ---还原---> DK+SS(8000x9000)---->|
                                               | --> 同类合并 --> 指标计算 --> 保留结果。
   LF（8000x9000） --------------------------->|
   ```

4. 合并计算时，进行了一些小优化，提升速度：对病害的 Xmin 进行排序，匹配时不用全部完全进行匹配；其次增加缓冲区，避免因为拼接导致的偏移致使一个病害变为 2 个。
5. 指标计算，指标计算主要就是涉及到裂缝的指标，DK+SS 就是求 BOX 面积然后进行映射，裂缝就主要涉及到最长的长度计算（BFS）以及延路径的最长宽度计算。当存在几米的病害时，python 会卡顿，进行了 C++ 重写，然后结合 pyhand 封装为 python 的可读取的版本。
6. 使用多线程进行计算、断点续写、报告输出等功能。
